# Этап 3 — классификация электротехнических работ и ветка 7.5

Реализация выполнена поверх архива этапа 2.

## Изменения runtime

- Добавлен `data/construction_work_dictionary_v6_4_6.json`.
- Runtime переключён на:
  - `DICTIONARY_SOURCE = construction_work_dictionary_v6_4_6`;
  - `PROMPT_VERSION = estimate-v6.4.6`;
  - `dictionary_version = construction_work_dictionary_v6_4_6@1.7.6`.
- JSON проходит `validate_dictionary_payload()` и `assert_project_hierarchy_compatible()`.

## Ветка 7.5

Добавлены этапы:

- `7.5.18` — кабеленесущие системы;
- `7.5.19` — внутреннее электроосвещение;
- `7.5.20` — наружное электроосвещение и управление;
- `7.5.21` — понижающие и разделительные трансформаторы.

Уточнены этапы ВРУ, распределительных щитов, кабельных линий, коробок, заземления, молниезащиты, электрообогрева, измерений и ПНР. Из электрической ветки удалены необоснованные связи `hvac` и `tv_radio`.

## StageClassifier

- Добавлен матчинг `detail_lines` как явного признака этапа.
- Баллы разделены в диагностике на:
  - `explicit_stage_evidence_score`;
  - `work_type_compatibility_score`;
  - `sequential_context_score`.
- Добавлена проверка уникальности `primary_work_type` внутри варианта.
- Совпадение общего `mep_internal/electrical` больше не снимает неоднозначность.
- При равных кандидатах порядок массива JSON не даёт автоматического принятия.
- Точный этап может уточнить тип работы: `ПНР` получает `mep_internal/commissioning`, даже если заголовок раздела дал общий электрический контекст.

## Row role и подсказка иерархии

- `Монтаж накладных светильников` определяется как работа, а не накладные расходы.
- `Накладные` и `Накладные расходы` остаются `overhead`.
- `ПНР` определяется как работа.
- Из скоринга подсказки исключены номера и внутренние id.
- Тип сметы рассчитывается по лучшему варианту, а не по сумме слабых совпадений всех вариантов.
- Для обеих контрольных смет первой подсказкой является `7 / 7.5`.
